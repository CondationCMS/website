Dokumentation

Alles, was du fuer ein ruhiges CMS-Fundament brauchst.

Von Installation bis Plugin-Erweiterung: eine kompakte Orientierung fuer moderne CondationCMS-Projekte.

Einleitung

CondationCMS ist fuer Teams gebaut, die Inhalte versionierbar, Templates nachvollziehbar und Erweiterungen wartbar halten wollen.

Installation

Installiere die Abhaengigkeiten und starte den Tailwind-Watcher fuer die lokalen Templates.

npm install
npm run build
npm run dev

Projektstruktur

Eine typische Seite trennt Content, Templates, Snippets und Assets klar voneinander.

source/
  snippets/
  test/
    src/
      index.html
      input.css
      layout.js

Content Modell

Collections beschreiben wiederkehrende Inhalte. Dadurch koennen Templates generisch bleiben und trotzdem fachlich praezise arbeiten.

Templates

Templates setzen Inhalte zusammen, statt Logik zu verstecken. Kleine Snippets sind dabei leichter zu testen und zu ersetzen.

Plugins

Plugins erweitern Verhalten ueber klare Hooks. So bleibt das CMS klein im Kern und stark an den Stellen, an denen Projekte es brauchen.
